# 论文审计规则 (Thesis-Audit)

## 触发指令

- 审计第X章 -> 对指定章节执行全部6项检查
- 审计公式 -> 仅执行维度1
- 审计引用 -> 仅执行维度2
- 审计格式 -> 仅执行维度3
- 审计文字 -> 仅执行维度4
- 审计逻辑 -> 仅执行维度5
- 审计论文 -> 全部维度逐章输出

## 公式输出规范（全局生效）

涉及公式的修改建议，必须用反引号(backtick)包裹$...$格式输出，确保显示为可直接复制的纯文本LaTeX源码，而非渲染后的数学符号。用户需要直接复制粘贴到MathType中。

正确输出方式（反引号包裹，显示纯文本源码）：
`$\hat{s} = y + r \cdot sk_{i} \textrm{ mod } q$`

MathType兼容性要求：
- 禁用多行环境（aligned/align/gather），每行公式单独一个反引号包裹的`$...$`
- 拼接符号用 \parallel 代替 \|
- 取模用 \textrm{ mod } 代替 \bmod
- 多行公式拆成多个单行，标注第1行、第2行

## 维度1：公式一致性（MathType对象校验）

检查范围：段间公式（Display Math，独立成行带编号）和段内公式（Inline Math，行内符号表达式）。

检查项：
1. 公式-上下文语义一致性：段间/段内公式与前后文描述是否匹配（检测MathType粘贴错位）
2. 变量定义一致性：每个变量是否有定义；同一符号全文含义一致
3. 段间-段内交叉一致性：行内写法与段间公式写法一致
4. 编号与引用：编号连续无跳号；如公式(X)引用指向正确
5. 常见粘贴错误：相邻公式重复；公式属于其他章节上下文

## 维度2：文献引用交叉比对

将正文[X]引用与文献列表逐一比对：
- 描述与该编号实际论文主题是否一致
- 有无幽灵文献（列了未引）或悬空引用（引了未列）
- 首次引用顺序与编号顺序一致
- 引用格式统一

## 维度3：页面格式

- 奇偶页页眉规范，翻章后更新
- 摘要罗马数字、正文阿拉伯数字页码连续
- 页眉字体字号下划线全文统一

## 维度4：文字表达与语言

- 错别字、多字、少字
- 口语禁用词：用->采用/引入；解决->规避/缓解；但是->然而；所以->鉴于此/进而；很多/很快->须量化
- 术语首次须标英文全称，全文用词统一
- 中英文标点混用

## 维度5：逻辑与结构

- 章节衔接：开头过渡段、结尾小结
- 绪论漏斗逻辑：背景->痛点->不足->方案
- 前后矛盾检测
- 图表引用与编号连续性

## 维度6：汇总报告

各维度问题统计（严重/中等/轻微）、TOP 10优先修改项、全文质量评价。
